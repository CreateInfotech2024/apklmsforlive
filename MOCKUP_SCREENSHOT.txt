    📡 Connected     (Green)  ← Socket.IO server connection active

    🟢 Green Dot   ← Peer connection established, streaming active
    🟠 Orange Dot  ← Connection in progress or reconnecting
    🔴 Red Dot     ← Connection failed (will retry automatically)

                            TECHNICAL DETAILS
════════════════════════════════════════════════════════════════════════════════

WebRTC Configuration:
  • ICE Servers: 5 STUN servers for better NAT traversal
  • SDP Semantics: Unified-plan for better compatibility
  • Audio: Echo cancellation, noise suppression, auto gain control
  • Video: 320-1280p resolution, 30 FPS max
  • Offer/Answer: Proper constraints for bidirectional streaming

Connection Flow:
  1. Participant joins meeting → Socket.IO connection
  2. Existing participants notified → Create WebRTC offer
  3. New participant receives offer → Creates answer
  4. ICE candidates exchanged → NAT traversal
  5. Peer connection established → Video/audio streams
  6. Status indicator updates → Green dot appears

Error Recovery:
  • Failed connections retry after 2 seconds
  • Network issues trigger automatic reconnection
  • Status indicators update in real-time
  • Console logs provide debugging information
